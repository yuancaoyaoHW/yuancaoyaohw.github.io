0一句话: 为什么需要 mHC?

💡 核心矛盾

Hyper-Connections (HC) 把残差流从 1 条拓宽到 n 条 (如 n=4), 让信息在不同深度的特征之间自由混合, 确实带来了性能提升。但 拓宽 + 自由混合 = 信号不守恒 — 跨层复合映射的增益可以爆炸到 ≈3000, 导致大模型训练时 loss 突然飙升、梯度发散。HC 用空间换性能, 却丢掉了 ResNet 十年来的立身之本: 恒等映射 (identity mapping)。

🎯 mHC 的解法

把 HC 里那个自由学习的残差映射矩阵 Hres, 通过 Sinkhorn-Knopp 算法投影到 双随机矩阵流形 (Birkhoff 多面体)上 — 强制每一行、每一列的和都等于 1。这样信号在流间混合时, 总能量守恒, 复合映射的增益被严格限制在 ≤1.6 (比 HC 的 3000 低三个数量级), 同时保留了多流混合的表达力。

📊 三种残差连接范式对比
ResNet: 严格恒等 → 稳定但无流间混合 | HC: 无约束 Hres → 混合但信号发散 | mHC: 双随机约束 → 既混合又守恒

1背景: 残差连接的十年

1.1 ResNet 奠基: 恒等映射的威力

2016 年 ResNet 确立了残差连接范式。单层传播为:

公式 (1) — 标准残差连接
xl+1 = xl + 𝓕(xl, 𝒲l)

其中 xl 是 C 维输入, 𝓕 是残差函数 (Attention、FFN 等)。这个形式之所以稳定, 是因为 恒等映射: 信号从浅层 l 直接传到深层 L 不被修改。递推展开:

公式 (2) — 多层递推 (恒等映射可见)
xL = xl + Σi=lL-1 𝓕(xi, 𝒲i)

注意等号右边的 xl 项: 它原封不动地从第 l 层出现在第 L 层。这就是"identity mapping"——浅层信号无衰减地直达任意深层。这条性质是 ResNet 能训练上百层网络的根本原因。

1.2 宏观设计的演进

过去十年, 微观设计 (块内部: 卷积→Attention+FFN, MQA/GQA/MLA, MoE) 进步巨大, 但宏观设计 (块间拓扑) 长期停滞在标准残差。近期出现一批工作试图拓宽残差流:

传统残差

  • 单流 (1×C)
  • 严格恒等映射
  • 稳定, 可训练性极强
  • 流间无信息交换

HC / DenseFormer / MUDDFormer

  • 多流 (n×C)
  • 可学习连接矩阵
  • 跨层跨流信息融合
  • 破坏恒等映射

mHC (本文)

  • 多流 (n×C)
  • 双随机约束矩阵
  • 跨层跨流信息融合
  • 恢复恒等映射

2HC 的工作原理: 拓宽残差流

Hyper-Connections (Zhu et al. 2024) 把残差流从 C 维扩展到 n×C 维 (n 为扩展率, 典型值 4)。它引入三个可学习映射:

📐 HC 的三个映射
  • Hprel ∈ ℝ1×n: 从 nC 维流中"读出" C 维送入层函数 𝓕
  • Hpostl ∈ ℝ1×n: 把 𝓕 的输出"写回" nC 维流
  • Hresl ∈ ℝn×n: 在残差流内部混合 n 条流 (最关键!)
公式 (3) — HC 单层传播
xl+1 = Hresl·xl + Hpost⊤l·𝓕(Hprel·xl, 𝒲l)

这三个映射由动态 (输入相关的, 经 RMSNorm + tanh 线性投影) 和静态 (可学习偏置) 两部分组成:

公式 (5) — HC 映射系数计算 (原始版本)
x̃l = RMSNorm(xl)
Hprel = αprel·tanh(θprel·x̃⊤l) + bprel
Hpostl = αpostl·tanh(θpostl·x̃⊤l) + bpostl
Hresl = αresl·tanh(θresl·x̃⊤l) + bresl
🧠 关键洞察: Hres 才是性能发动机

论文 Table 1 的消融实验表明: 禁用 Hpre 或 Hpost 性能损失不大, 但禁用 Hres 性能下降最显著。这说明 HC 的收益主要来自残差流内部的跨流信息交换, 而非单纯的维度扩展。但正是这个 Hres, 也成了不稳定的根源。

3HC 的两大病灶

3.1 病灶一: 数值不稳定 — 复合映射爆炸

把 HC 的单层公式 (3) 跨多层递推, 从层 l 到层 L:

公式 (4) — HC 多层复合映射 (问题所在!)
xL = (∏i=1L-l HresL-i)·xl + Σi=lL-1 (∏j=1L-1-i HresL-j)·Hpost⊤i·𝓕(...)
⚠️ 问题核心

对比 ResNet 的公式 (2), 那里 xl 前的系数是 1 (恒等)。而 HC 这里, xl 前的系数变成了矩阵乘积 ∏ Hres。由于 Hres 是无约束的可学习矩阵, 这个乘积的谱范数完全不可控 —— 信号会指数级放大或衰减。

论文用 Amax Gain Magnitude (复合映射行/列和绝对值的最大值) 量化这个问题。实测在 27B 模型中:

📈 HC vs mHC 信号传播对比
左: HC 复合映射 ∏Hres 增益指数爆炸 (峰值≈3000) | 右: mHC 复合映射有界 (峰值≈1.6)

实测现象: HC 在 27B 训练到约 12k 步时loss 突然飙升, 与梯度范数异常高度相关 (论文 Figure 2)。复合映射的 Amax Gain 峰值达到 3000 — 远离理想值 1 整整三个数量级, 确认残差流发生了爆炸。

📉 复合映射增益 vs 层深度 (对数尺度)
HC (橙): 随层深度指数增长至 ≈3000 | mHC (蓝): 始终有界, 峰值仅 ≈1.6, 降低三个数量级

3.2 病灶二: 系统开销 — 内存墙与通信膨胀

🧱 内存墙 (Memory Wall)

HC 的 FLOPs 几乎不增加 (映射矩阵很小), 但内存访问 (I/O) 成本按 n 倍增长。在现代架构中, I/O 往往是真正瓶颈 ("memory wall"), 而非计算。论文 Table 2 分析: 单层残差维护的 per-token 内存访问, HC 约为标准残差的 n 倍。

具体来说, HC 带来三重系统负担:

  1. 前向 I/O 膨胀: n 条流都要读写, 内存访问量 ∝ n
  2. 反向显存膨胀: Hpre、Hpost、Hres 的中间激活都需保存用于反向, 显存占用大增, 常需梯度检查点
  3. 流水线通信膨胀: 流水线并行 (PP) 中, 激活在 stage 间传递, n 条流意味着 n 倍通信量, 气泡变大, 吞吐下降

4mHC 核心思想: 流形投影恢复恒等映射

💡 设计哲学

ResNet 的恒等映射要求 Hres = I (单位矩阵) — 稳定但完全禁止流间混合, 浪费了多流架构的潜力。mHC 的洞见: 我们不需要 Hres 恒等于 I, 只需要它落在某个保证信号守恒的流形上。这样既能混合 (表达力), 又能守恒 (稳定性)。

mHC 选择的流形是 双随机矩阵集合 (又称 Birkhoff 多面体)。形式化定义:

公式 (6) — Birkhoff 多面体 (双随机矩阵流形)
𝒫𝓜res(Hresl) := { Hresl ∈ ℝn×n | Hresl·𝟏n = 𝟏n, 𝟏⊤n·Hresl = 𝟏⊤n, Hresl ≥ 0 }

即: 矩阵所有元素非负, 且每一行、每一列的和都等于 1。当 n=1 时退化为标量 1, 正好恢复原始恒等映射。

🔷 Birkhoff 多面体: 双随机矩阵的几何意义
无约束的 Hres (橙点) 从全空间被投影到 Birkhoff 多面体 (绿域), 成为双随机矩阵 P𝓜(Hres)

4.1 为什么双随机矩阵能解决一切?

双随机约束带来 四个关键性质, 恰好对应 HC 的所有病灶:

① 范数保持 (Norm Preservation)

双随机矩阵的谱范数 ≤ 1 (即 ‖Hresl‖2 ≤ 1)。这意味着每一层的映射是非膨胀的 (non-expansive) — 信号通过它不会被放大。这从根本上消除了梯度爆炸的风险。

② 组合闭合 (Compositional Closure)

双随机矩阵集合在矩阵乘法下封闭: 两个双随机矩阵相乘, 结果仍是双随机矩阵。因此跨多层复合 ∏ HresL-i 仍然是双随机的, 谱范数仍 ≤ 1。整条残差路径从头到尾都守恒, 不论模型多深。

⚡ 这是 mHC 最漂亮的一点

ResNet 的恒等映射 I 也是一种"守恒", 但它是平凡的守恒 (不混合)。双随机矩阵提供了非平凡的守恒: 信号在 n 条流之间重新分配 (像概率重分布), 但总量不变。这正好兼顾了"混合"和"守恒"两个看似矛盾的目标。

③ 几何意义: 排列矩阵的凸组合

Birkhoff 多面体是所有排列矩阵的凸包。也就是说, 任何双随机矩阵都可以写成若干排列矩阵 Pk 的加权平均: Hres = Σ λk·Pk (λk ≥ 0, Σλ = 1)。操作 Hres·x 相当于对输入做若干种"打乱"再加权融合 — 一种天然的特征融合机制。而且反复施加这类矩阵会让信息在流间趋于均匀混合。

④ 信号均值守恒

由于行和为 1, Hres·x 的每个输出是输入的凸组合, 输出均值 = 输入均值。流间信号强度的平均值在每层保持不变, 从根本上杜绝了某条流爆炸而其他流枯竭的不均衡现象。

4.2 参数化与流形投影

mHC 的实际计算流程: 先用 HC 的方式生成"自由"的映射系数, 再用约束算子把它们投影到目标流形。

第一步: 展平输入, 经 RMSNorm + 线性投影得到自由的 H̃pre, H̃post, H̃res (与公式 5 类似但用展平输入):

公式 (7) — 自由映射系数
x⃗'l = RMSNorm(x⃗l)
H̃prel = αprel·(x⃗'l·φprel) + bprel
H̃postl = αpostl·(x⃗'l·φpostl) + bpostl
H̃resl = αresl·mat(x⃗'l·φresl) + bresl

第二步: 对自由系数施加约束算子, 得到最终的映射:

公式 (8) — 流形投影后的最终映射
Hprel = σ(H̃prel)   (sigmoid 保证非负)
Hpostl = 2σ(H̃postl)   (2σ 保证非负且均值为1)
Hresl = Sinkhorn-Knopp(H̃resl)   (投影到双随机流形!)
📝 设计细节
  • Hpre 用 sigmoid → 非负, 防止正负系数相消导致信号抵消
  • Hpost 用 2σ → 非负且期望值约为 1 (σ 输出 [0,1], 乘 2 后 [0,2], 均值≈1)
  • Hres 用 Sinkhorn-Knopp → 双随机 (核心约束)

5Sinkhorn-Knopp 算法: 如何投影到双随机流形

给定一个任意实数矩阵, 如何找到"最近"的双随机矩阵? Sinkhorn-Knopp 算法是经典解法: 先取指数让所有元素变正, 然后交替做行归一化和列归一化, 迭代收敛。

公式 (9) — Sinkhorn-Knopp 迭代
M(0) = exp(H̃resl)   (先取指数, 保证全正)
M(t) = 𝒯r(𝒯c(M(t-1)))   (交替行/列归一化)
Hresl = M(tmax)   (tmax→∞ 时收敛到双随机)

其中 𝒯r 是行归一化 (每行除以行和), 𝒯c 是列归一化。这个交替过程保证收敛到双随机矩阵。论文实际用 tmax = 20 次迭代作为近似解。

🔄 Sinkhorn-Knopp 迭代过程 (4×4 矩阵, 20 步)
矩阵元素随迭代逐步调整, 行和与列和交替收敛到 1, 最终成为双随机矩阵
⚠️ 近似解的代价与补偿

20 次迭代只是近似双随机, 不完全精确。实测复合映射的反向梯度增益会略有偏差 (单层≈1, 复合后峰值约 1.6)。但相比 HC 的 3000, 这个偏差小了三个数量级, 完全可接受。这是"计算效率"与"约束精度"的务实折中。

6基础实施优化: 让 n=4 只多花 6.7%

💡 为什么需要这一章?

mHC 的数学很优雅, 但如果 n=4 让训练慢 4 倍, 没人会用。论文花大力气做系统工程, 最终把额外开销压到仅 6.7%。这证明 mHC 不只是理论漂亮, 而是可工业部署的。

6.1 Kernel Fusion (算子融合)

用 TileLang 框架实现三个专用 fused kernel:

  • Kernel 1 (Eq.14-15): 把对 x⃗l 的两次扫描合并, 利用矩阵乘单元最大化带宽。反向的两个矩阵乘也合成单 kernel, 避免重复加载 x⃗l。含精细的 load→cast→compute→store 流水线处理混合精度
  • Kernel 2 (Eq.16-18): 轻量系数操作融合成单 kernel, 减少启动开销
  • Kernel 3 (Eq.19): Sinkhorn-Knopp 迭代在单 kernel 内完成; 反向用自定义 kernel 在片上重算整个迭代

还有两个应用 kernel: 𝓕pre = Hpre·x 和 𝓕post,res = Hres·x + Hpost⊤·𝓕。通过把 Hpost 和 Hres 的应用与残差合并融合, 该 kernel 的读取量从 (3n+1)C 降到 (n+1)C, 写入从 3nC 降到 nC。

RMSNorm 重排技巧: 原本 RMSNorm 在高维 x⃗l ∈ ℝ1×nC 上操作很慢。mHC 把"除以范数"挪到矩阵乘之后做, 数学等价但更快。同时把 RMSNorm 权重吸收进投影矩阵 φ。

6.2 选择性 Recomputing (重计算)

n 条流的中间激活显存巨大。mHC 的策略: 前向后丢弃 mHC kernel 的中间激活, 反向时重新计算 (但不重算重的 𝓕 层函数)。对于连续 Lr 层的块, 只需存第一层输入 xl0。

最优块大小 Lr* 由总显存最小化决定:

公式 (20) — 最优重计算块大小
Lr* = argminLr [ nC × ⌈L/Lr⌉ + (n+2)C × Lr ] ≈ √(nL / (n+2))

第一项是所有块的常驻显存 (每块存 nC), 第二项是当前活跃块的瞬时显存 ((n+2)C × Lr)。两者平衡给出最优 Lr* ≈ √(nL/(n+2))。实测这个值正好与流水线 stage 的层数对齐, 所以把重计算边界与流水线 stage 对齐。

6.3 DualPipe 通信重叠

DeepSeek-V3 的 DualPipe 调度能重叠专家并行和流水线并行的通信。但 mHC 的 n 条流在 stage 边界引入额外通信延迟, 且 stage 边界要重算 Lr 层的 mHC kernel。论文扩展 DualPipe 来重叠这些开销:

🌊 mHC 完整前向数据流 + 基础设施优化
从 xl 到 xl+1 的完整数据流, 含三路映射、层函数 𝓕、残差合并, 以及三大基础设施优化
  • 专用高优先级流: MLP 层的 𝓕post,res kernel 在专用高优先级计算流上执行, 防止阻塞通信流
  • 避免 persistent kernel: Attention 层不用长时 persistent kernel, 防止长时间占用导致通信 stall, 允许灵活抢占调度
  • 重计算与通信解耦: 每个 stage 的初始激活 xl0 已本地缓存, 重计算不依赖流水线通信

8总结与展望

💡 核心贡献

本文识别出 HC 的根本问题: 无约束连接破坏信号能量守恒, 导致训练不稳定和扩展性受限。mHC 通过把残差映射投影到双随机矩阵流形, 把信号传播变成特征的凸组合, 恢复了恒等映射的守恒性质, 同时保留了多流混合的表达力。配合 kernel fusion + 重计算 + DualPipe 通信重叠, 实现仅 6.7% 额外开销的工业级部署。

8.1 方法论启示

mHC 的思路有普适价值: 不要在"自由"和"恒等"之间二选一, 而是找一个流形, 让自由落在守恒的子空间里。这调和了 ResNet 的稳定性和 HC 的表达力这对矛盾。

8.2 未来方向

  • 其他流形约束: 双随机只是其中一种选择, 可探索针对不同学习目标的几何约束, 在可塑性与稳定性之间寻找更优折中
  • 宏观架构设计复兴: mHC 希望重燃社区对拓扑结构设计的兴趣 — 过去十年大家都盯着块内部, 块间拓扑还有大量空间
  • 拓扑如何影响优化与表示学习: 深入理解结构如何影响梯度流和表征形成, 可能照亮下一代基础架构的演化路径
