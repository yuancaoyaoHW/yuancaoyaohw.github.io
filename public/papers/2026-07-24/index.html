Windowed-MTP: Removing the Full-Context Draft-KV Tax at Million-Token Context

⭐⭐⭐⭐☆ (4/5)
作者: Alagappan Valliappan  |  机构: NVIDIA  |  分类: cs.LG  |  日期: 2026-07-24
arXiv · PDF
Speculative decoding 通过让廉价的 draft head 提出候选 token、再由 target 模型并行验证来加速自回归生成。前沿模型普遍内置 MTP/NEXTN draft head,前提是 draft 成本可忽略。但在百万 token 上下文下这一假设崩塌:MTP draft head 每步都要对整个 KV cache 做 full attention,读取量随上下文线性增长。本文对 draft attention 施加 StreamingLLM 式 sliding window + attention sink,验证保持 full attention 不变,training-free 且构造上无损。在三大架构族上 per-decode-step 成本降低 28%-44%,增益随上下文增长。

核心贡献

  • 识别并量化百万 token 上下文下 MTP draft head 的'全上下文 draft-attention 税':1M 时额外增加 +92% 到 +138% 开销
  • 提出 Windowed-MTP:training-free sliding window + attention sink(W=4032),验证保持 full attention,构造上无损
  • 跨三大架构族(Qwen GDN-MoE、Mamba2-hybrid、NoPE)验证一致加速,未读取 draft KV 通过 ring buffer 回收

主要结论

  • 内置 MTP draft head 在百万 token 下可使深 speculation 慢于无 speculation
  • Windowed-MTP 以 drop-in、保持 target 分布的方式消除该成本,端到端从不回退
  • 增益随上下文增长(windowing wedge widens),对 hybrid/linear-attention target 尤其有效

实验结果

  • 单卡 NVIDIA B200(TP1),bf16 KV,SGLang 引擎+CUDA graphs;模型 Qwen GDN-MoE 35B/122B、Mamba2-hybrid 120B
  • 1M 上下文(γ=6)时 draft 阶段额外 +92% 到 +138% 开销,近乎使 decode 步成本翻倍
  • Windowed-MTP 将 draft KV 工作集限制为常数(W=4032),丢弃约 99% KV 条目
  • per-decode-step 成本降低 28%-44%,end-to-end decode 延迟同幅改善
  • greedy output 与原生 MTP 在 bf16 非确定性范围内 bit-exact(lossless)
为什么重要: 揭示了 MTP speculative decoding 在百万 token 上下文下的根本性瓶颈并给出 training-free 无损解法,对 SGLang/vLLM 等推理引擎在长上下文场景的落地有直接价值

PolyQ: Codesigning End-to-End Quantization Framework for Scalable Edge CPU LLM Inference

⭐⭐⭐⭐☆ (4/5)
作者: Hyunwoo Oh, Suyeon Jang, Hanning Chen  |  机构: UC Irvine  |  分类: cs.OS  |  日期: 2026-07-24
arXiv · PDF
CPU 是端侧 LLM 推理最通用的目标,但现有低比特量化方法要么提供粗粒度操作点,要么提供难以在 CPU 上高效执行的细粒度混合精度。PolyQ 是面向 CPU 的编译器/量化协同设计,在用户指定的平均比特预算下做 activation-aware 的 channel-wise 比特分配,从 {2,3,4,8,16} 中分配比特宽度,用编译期模型编译器将 channel 重排聚簇为 bit-homogeneous 块、生成 SIMD/LUT 兼容 kernel。在 Falcon-H1-3B、Llama2-13B、Qwen3-32B 上 perplexity 改善 2.4%-32.1%。

核心贡献

  • 可部署的 channel-wise 混合精度量化方案:activation-aware saliency + {2,3,4,8,16}-bit 调色板,支持 0.1-bit 步长 fractional-bit 适配
  • compile-time 量化-编译器协同设计:permutation 聚簇 + SIMD/LUT kernel 生成 + 算子间 permutation 传播,layout 正则化移出 runtime
  • 端到端验证 budget 可扩展性,弥合 coarse inter-layer 与 fine intra-layer 混合精度之间的工程鸿沟

主要结论

  • Fractional-bit CPU 部署可行、可预测、能效友好——同一预算参数同时控制峰值显存并给出单调 latency/throughput 曲线
  • 编译器 layout 正则化将 activation reorder 流量降低高达 70.8%,使细粒度混合精度在 CPU 上实际可执行
  • Prefill 延迟与 decode 吞吐随比特预算近乎成比例缩放,energy/token 开销 <2%

实验结果

  • 模型:Falcon-H1-3B、Llama2-13B、Qwen3-32B;数据集 WikiText-2;硬件:工作站/笔记本/移动 CPU
  • 3b 目标处 perplexity 较 AWQ/GPTQ/Slim-LLM/AMQ 改善 2.4%-32.1%
  • 编译器 layout 正则化使 activation reorder 流量降高达 70.8%
  • Prefill 延迟与 decode 吞吐随比特预算近乎成比例缩放;energy/token <2% 相对优化 LUT 后端
  • 2→3b 延迟升 1.56×,3→4b 升 1.38-1.46×(接近理想流量比);B=3 能耗/token 开销 0.8-1.9%
为什么重要: 首次让 fractional-bit 混合精度量化在 CPU 上实际可部署且可预测,弥合了 coarse inter-layer 与 fine intra-layer 量化之间的工程鸿沟

BaseRT: Advancing Best-in-Class LLM Inference with Apple M5 Neural Accelerators

⭐⭐⭐⭐☆ (4/5)
作者: Fabian Waschkowski, Prabod Rathnayaka, Lukas Wesemann  |  机构: Base Compute, Melbourne, Australia  |  分类: cs.DC  |  日期: 2026-07-24
arXiv · PDF
Apple M5 每核心携带专用 Neural Accelerator(通过 Metal 4 tensor API 暴露的片上矩阵单元)。BaseRT 是 Apple Silicon 上 LLM 的原生 Metal 推理运行时,利用这些单元将推理吞吐推至大幅超越 llama.cpp 和 MLX。新增手写 Metal 4 tensor-core kernel(含 dense 和 MoE GEMM 及 flash-attention prefill kernel),将计算受限矩阵乘法路由到 M5 Neural Accelerators。在 M5 Pro 上跨 Qwen3/Llama 3.2/Gemma 4 共 15 种配置,prefill 吞吐较 llama.cpp 高达 6.4×。

核心贡献

  • 手写 Metal 4 协同 tensor-core kernel 系列,直接驱动 Apple M5 每核心 GPU 神经加速器(含 dense GEMM、MoE 专家 GEMM、融合专家投影、prefill attention)
  • 工作负载感知调度策略:仅计算受限阶段走 tensor-core,内存受限 decode 留专用 kernel,不满足约束或 M5 之前芯片平滑回退至 SIMD
  • 跨 Qwen3/Qwen3.5/3.6/Llama 3.2/Gemma 4 系列(0.6B-35B)15 种配置全面评估,确立新 SOTA 端侧 LLM 吞吐

主要结论

  • Apple M5 逐核心 Neural Accelerator 是 Apple Silicon 上 prefill 吞吐的决定性杠杆,相对 llama.cpp 领先高达 6.4×,相对 MLX 高达 3.9×
  • MoE 模型优势最大(矩阵乘法 per active parameter 占比高),decode 路径内存受限故增益较小但仍领先
  • 仅两处例外 MLX 领先(Llama-3.2-1B/3B 长 context),幅度 ≤3%

实验结果

  • 硬件:Apple M5 Pro(48GB 统一内存);模型 Qwen3/Qwen3.5/3.6/Llama 3.2/Gemma 4,0.6B-35B,15 种配置(含 4 MoE)
  • Prefill 吞吐:较 llama.cpp 高达 6.4×,较 MLX 高达 3.9×;最大边际在 MoE 模型(Qwen3-30B-A3B pp128)
  • Decode 吞吐:较 llama.cpp 高达 1.75×,较 MLX 高达 1.33×
  • MLX 仅在 Llama-3.2-1B/3B 长 context(pp1024-2048)领先 ≤3%
  • 10 种 4-bit 量化,5 种较小模型 8-bit;基线 llama.cpp b9960 + mlx-lm 0.31.3
为什么重要: 确立了 Apple M5 Neural Accelerator 在端侧 LLM 推理中的决定性地位,对 Apple Silicon 上的推理引擎设计有直接工程指导价值

GradRAG: Cross-Component Prompt Adaptation for Coordinated Multi-Agent RAG

⭐⭐⭐⭐☆ (4/5)
作者: Paolo Pedinotti, Enrico Santus  |  机构: Bloomberg  |  分类: cs.CL  |  日期: 2026-07-24
arXiv · PDF
将 RAG 流水线建模为计算图,Evaluator 评判下游答案与证据并产出可执行反馈,Prompt Optimizer 据此迭代更新多个上游 agent(检索器/图构造器/答案器)的提示。支持早停。两种检索范式:flat retrieval 和 graph-based retrieval。GradRAG 将 RAG 视为多组件系统,通过跨组件梯度式提示适配实现端到端协调优化,无需重新训练或微调模型权重。

核心贡献

  • 将 RAG 流水线建模为计算图,实现跨组件梯度式提示适配(Evaluator→Prompt Optimizer→上游 agents),无需重训模型权重
  • 支持两种检索范式:flat retrieval 和 graph-based retrieval,统一框架适配不同 RAG 架构
  • 早停机制:当 Evaluator 判定答案质量达标时提前终止迭代,降低 token 成本

主要结论

  • 跨组件提示适配可有效协调多 agent RAG 系统,无需代价高昂的模型微调
  • Evaluator 的可执行反馈是驱动上游优化的关键,闭环优化优于单点调优
  • 框架对 flat 和 graph-based 两种检索范式均有效

实验结果

  • 评估 flat retrieval 和 graph-based retrieval 两种范式
  • Evaluator 评判答案质量与证据支撑,产出可执行反馈驱动 Prompt Optimizer
  • 跨组件迭代优化:检索器/图构造器/答案器的提示同步适配
  • 早停机制在答案达标时提前终止,降低 token 开销
  • 机构为 Bloomberg(Pedinotti 实习期间完成)
为什么重要: 首个将 RAG 建模为计算图并实现跨组件梯度式提示适配的框架,为多 agent RAG 系统的端到端协调优化提供了新范式

核心贡献

  • Modulus-aware bucket checksum:利用宽 CPU 累加器(AVX-512)批量缩减 checksum 乘积,替代逐系数校验
  • Dataflow-fused in-operator checking:将 checksum 累积直接嵌入算子数据流,避免独立校验遍历
  • 将容错开销从 O(n²) 降至近线性,使 CPU 上 CKKS 的容错检查实际可行

主要结论

  • 通过模感知桶 checksum 和数据流融合,将 CPU 上 CKKS 容错开销降至近线性
  • 宽 CPU 累加器(AVX-512)是降低 modular reduction 成本的关键硬件杠杆
  • 嵌入式校验避免了独立遍历的额外内存带宽压力

实验结果

  • 在 CPU 上评估 CKKS 全同态加密计算的容错方案
  • 利用 AVX-512 宽累加器批量缩减 checksum 乘积
  • 将 checksum 累积嵌入算子数据流,避免独立校验遍历
  • 容错开销从 O(n²) 降至近线性
  • 保证计算正确性前提下的开销对比
